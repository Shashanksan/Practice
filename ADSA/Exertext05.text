Problem Statement:



Nisha is working on a Python project to evaluate complex mathematical expressions written in postfix notation (also called Reverse Polish Notation). She wants to simulate how a compiler parses and evaluates expressions like (3 + (5 * 2)) using a binary expression tree.

To do this, she decides to:

Construct an Expression Tree, where:
Each operand becomes a leaf node.
Each operator becomes an internal node.
Perform a post-order traversal to evaluate the expression from the bottom up.


Help Nisha write a Python program that:

Builds an expression tree from a postfix input string.
Evaluates the expression using post-order traversal.
Prints the result or shows an error message if the input is invalid
Input format :
A single line containing space-separated tokens of a postfix (Reverse Polish Notation) expression.

Tokens can be:

Integers (operands)
Operators: +, -, *, /
Output format :
The output displays the following:

If the expression is valid: Output: <result>
If the expression is invalid (due to insufficient operands or unrecognized symbols), print: Invalid expression.
If a division by zero is encountered during evaluation, print: Output: Division by zero


Refer to the sample output for formatting specifications.

Sample test cases :
Input 1 :
3 5 2 * +
Output 1 :
Output: 13
Note :
The program will be evaluated only after the “Submit Code” is clicked.
Extra spaces and new line characters in the program output will result in the failure of the test case.
