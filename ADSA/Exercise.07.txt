ingle File Programming Question
Problem Statement:



Deepak, a graduate student working on a Python-based student ID management system, wants to efficiently store and retrieve student IDs using hashing. However, since multiple IDs might hash to the same index, he decides to explore collision resolution techniques.

To handle such collisions, Deepak implements two classic methods:

Linear Probing – where the algorithm searches linearly for the next available slot.
Quadratic Probing – where the algorithm searches in quadratic steps to reduce clustering.


Your task is to help Deepak develop a Python program that:

Inserts student IDs into a hash table using either linear or quadratic probing.
Searches for a student ID using the chosen probing technique.
Displays the current state of the hash table.
Handles invalid choices and edge cases gracefully
Input format :
Each line of input represents a command in the following format:



1 <ID> <method> → Insert student ID

<ID>: Integer value representing the student ID
<method>: 1 for Linear Probing, 2 for Quadratic Probing


2 <ID> <method> → Search for a student ID

<ID>: Integer value to search
<method>: 1 for Linear Probing, 2 for Quadratic Probing


3 → Display the current state of the hash table



4 → Exit the program

Output format :
The output displays the following:

1. On successful insertion: Value <ID> inserted successfully using Linear Probing or Value <ID> inserted successfully using Quadratic Probing.

2. On insertion failure: Hash table is full. Insertion failed.

3. On search:

If found: Value <ID> found at index <index> using Linear Probing. or Value <ID> found at index <index> using Quadratic Probing.
If not found: Value <ID> not found using Linear Probing or Value <ID> not found using Quadratic Probing.
4. On display: <space-separated list of 10 elements in the hash table, -1 for empty slots>

5. On invalid input: Invalid probing method. or Invalid choice.
